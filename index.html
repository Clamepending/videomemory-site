What it is

VideoMemory is a small server that lets an agent create and manage video monitoring tasks (e.g. tell me when the cat is licking its paw, text me when the snow is high enough to call the shovelers) through natural language. It supports built-in cameras, USB cameras, browser camera streams, and RTSP streams (streaming from your phone). Agents interact with it through HTTP.

Put the visual trigger condition in VideoMemory. Your agent should own the action sequence before and after the trigger. So if you wanted an agent to restock your pantry when it is running low, your agent would be reposnsible for specifying what "running low" means, knowing what you like, and actually ordering food. Videomemory would be responsible for emitting a callback to your agent when the pantry is detected low via a video stream.

Quickstart

git clone https://github.com/Clamepending/videomemory
cd videomemory
uv sync
uv run videomemory-server

Then open http://127.0.0.1:5050, set a model API key in Settings, choose a camera, and create a task.

Create a monitor

curl -X POST http://127.0.0.1:5050/api/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "io_id": "browser_facetime",
    "task_description": "A phone is visible in the frame.",
    "bot_id": "my-agent",
    "save_note_frames": true,
    "save_note_videos": true
  }'

The example in examples/voice-agent-demo uses a browser camera, microphone, OpenAI Realtime voice, and VideoMemory monitors. While you talk, the agent can create one-shot or persistent visual wakeups. 

cd examples/voice-agent-demo

Example tasks

  • Tell me when a phone is visible.
  • Each time I hold up fingers, add the number to a running total.
  • Be a shopkeeper. Watch these apples and keep a ledger.
  • Wake my agent when someone enters the room.

HTTP endpoints

  • GET /api/devices lists available inputs.
  • POST /api/caption_frame asks about the current frame.
  • POST /api/tasks creates a monitor.
  • GET /api/task/{task_id} reads notes and evidence.
  • POST /api/task/{task_id}/stop stops a monitor.

How it works

  1. The camera ingestor samples frames.
  2. The model checks the active task descriptions.
  3. VideoMemory saves task notes, frames, and optional clips.
  4. A webhook wakes the agent with the task update.

Agent quickstarts

Give your coding agent the small onboarding file instead of explaining the API from scratch. Use llms.txt for LLM-friendly context, or AGENTS.md when your coding tool reads repository agent instructions.

Coding agents

Point the agent at the local server, then have it list devices and create monitor tasks through HTTP. 

curl -fsSL https://clamepending.github.io/videomemory-site/llms.txt

Claude Code

Use the packaged launcher for local VideoMemory setup, the camera bridge, the Claude channel, and wakeup-capable monitor tools.
