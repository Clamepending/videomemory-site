Open-source toolkit for video-aware agents

VideoMemory

Build agents that can keep watching after the chat turn ends. Give VideoMemory a camera feed and a natural-language condition; it saves evidence and wakes your app when the visual event happens.

VideoMemory is deliberately small at the integration boundary. Run the local server, create a visual task, and point webhooks at your own agent process.

Local quickstart

git clone https://github.com/Clamepending/videomemory
cd videomemory
uv run flask_app/app.py

Open the local UI, add a model key, choose a camera, and create a task.

Use the voice-agent example

cd examples/voice-agent-demo
OPENAI_API_KEY=sk-... npm start

Use this as the template for your own tool-calling video agent.

Connect your webhook

PUT /api/settings/VIDEOMEMORY_OPENCLAW_WEBHOOK_URL
POST /api/tasks
GET /api/task/{task_id}

Keep the action in your app. Put only the visual condition in VideoMemory.

How it works

VideoMemory stays in the perception loop. Your agent stays in the conversation loop.

  1. 01

    Connect a stream

    Use a built-in camera, USB webcam, RTSP stream, Android phone, or browser camera bridge.

  2. 02

    Create a monitor

    Post a natural-language task such as “a phone is visible” or “a person enters.”

  3. 03

    Analyze changes

    The ingestor samples frames, asks a VLM for task updates, and saves note evidence.

  4. 04

    Wake the agent

    Webhook payloads include the task, note, frame URL, and video clip URL.

Developer primitives

Everything useful is exposed as boring HTTP.

Event-driven monitors

Agents create visual conditions once and wait for webhook wakeups instead of polling frames.

Evidence-first debugging

Task notes can preserve the triggering frame and a short video clip for replay.

Plain HTTP control plane

Device discovery, tasks, settings, captures, and frame captions are all available by API.

API

Create a monitor with one request.

Put the visual condition in VideoMemory. Keep the user-facing action in the agent runtime. 

curl -X POST http://127.0.0.1:5050/api/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "io_id": "browser_facetime",
    "task_description": "A phone is visible in the frame.",
    "bot_id": "my-agent",
    "save_note_frames": true,
    "save_note_videos": true
  }'
