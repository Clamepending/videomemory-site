What it is

VideoMemory is a small server for natural-language video monitoring tasks. Ask it to watch for things like a cat licking its paw, snow high enough to call shovelers, or someone showing interest in products on a table.

It supports built-in cameras, USB cameras, browser camera streams, and RTSP streams from phones or network cameras. Agents interact with it through HTTP.

Put the visual trigger condition in VideoMemory. Your agent owns the action sequence before and after the trigger. For a pantry-restocking agent, VideoMemory detects the low-pantry state; the agent decides what "running low" means and handles the order.

Quickstart

Start the local service, then open the browser UI to choose a camera and create a task. 

git clone https://github.com/Clamepending/videomemory
cd videomemory
uv sync
uv run videomemory-server

Then open http://127.0.0.1:5050, set a model API key in Settings, choose a camera, and create a task.

Create a monitor

A monitor is just an input device plus a plain-English condition. 

curl -X POST http://127.0.0.1:5050/api/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "io_id": "browser_facetime",
    "task_description": "A phone is visible in the frame.",
    "bot_id": "my-agent",
    "save_note_frames": true,
    "save_note_videos": true
  }'

Voice agent demo

The example in examples/voice-agent-demo uses a browser camera, microphone, OpenAI Realtime voice, and VideoMemory monitors. While you talk, the agent can create one-shot or persistent visual wakeups. 

cd examples/voice-agent-demo
OPENAI_API_KEY=sk-... npm start

Example tasks

  • Presence: tell me when a phone is visible.
  • Counting: each time I hold up fingers, add the number to a running total.
  • Sales assistant: when someone interacts with products on the table, say hi and try to sell them.
  • Automation: when someone enters the room, turn on the lights and start the coffee machine.

HTTP endpoints

  • GET /api/deviceslists available inputs.
  • POST /api/caption_frameasks about the current frame.
  • POST /api/taskscreates a monitor.
  • GET /api/task/{task_id}reads notes and evidence.
  • POST /api/task/{task_id}/stopstops a monitor.

How it works

  1. SampleThe camera ingestor samples frames.
  2. CheckThe model checks the active task descriptions.
  3. RecordVideoMemory saves task notes, frames, and optional clips.
  4. WakeA webhook wakes the agent with the task update.
