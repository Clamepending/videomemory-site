What it is

VideoMemory is a small server that lets an agent create and manage video monitoring tasks. It supports built-in cameras, USB cameras, browser camera streams, and RTSP streams. Agents interact with it through HTTP.

Put the visual condition in VideoMemory. Keep the action in your agent: speak to the user, update a ledger, send a message, call a robot policy, or ignore the event.

Quickstart

git clone https://github.com/Clamepending/videomemory
cd videomemory
uv run flask_app/app.py

Then open http://127.0.0.1:5050, set a model API key in Settings, choose a camera, and create a task.

Create a monitor

curl -X POST http://127.0.0.1:5050/api/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "io_id": "browser_facetime",
    "task_description": "A phone is visible in the frame.",
    "bot_id": "my-agent",
    "save_note_frames": true,
    "save_note_videos": true
  }'

Voice agent demo

The example in examples/voice-agent-demo uses a browser camera, microphone, OpenAI Realtime voice, and VideoMemory monitors. While you talk, the agent can create one-shot or persistent visual wakeups. 

cd examples/voice-agent-demo
OPENAI_API_KEY=sk-... npm start

Example tasks

  • Tell me when a phone is visible.
  • Each time I hold up fingers, add the number to a running total.
  • Be a shopkeeper. Watch these apples and keep a ledger.
  • Wake my agent when someone enters the room.

HTTP endpoints

  • GET /api/devices lists available inputs.
  • POST /api/caption_frame asks about the current frame.
  • POST /api/tasks creates a monitor.
  • GET /api/task/{task_id} reads notes and evidence.
  • POST /api/task/{task_id}/stop stops a monitor.

How it works

  1. The camera ingestor samples frames.
  2. The model checks the active task descriptions.
  3. VideoMemory saves task notes, frames, and optional clips.
  4. A webhook wakes the agent with the task update.
